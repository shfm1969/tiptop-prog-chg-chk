## Context

`chklog.py` 的 `main()` 函式在第 158-160 行開啟輸出檔案，目前只寫入標題和產生時間。三個輸入檔案路徑已定義在全域變數 `EXCEL_PATH`、`TXT_4GL_4FD`、`TXT_RPT_XML` 中。

## Goals / Non-Goals

**Goals:**
- 在報告開頭（標題區塊之後、第一個檢查結果之前）加入輸入資料來源段落
- 顯示每個輸入檔案的檔名與最後修改日期

**Non-Goals:**
- 不改變既有的檢查邏輯
- 不驗證檔案內容的正確性
- 不改變輸出檔案的命名規則

## Decisions

**決策 1：使用 `os.path.getmtime()` 取得檔案修改日期**

- 理由：Python 標準庫即可取得，無需額外依賴
- 替代方案：讀取檔案內容中的日期 → 太複雜且不通用
- 格式化為 `YYYY-MM-DD HH:MM:SS` 與現有報告時間格式一致

**決策 2：在現有標題區塊之後、檢查結果之前插入**

- 理由：這是最自然的位置，讓使用者先看到來源資訊再看結果
- 使用與現有分隔線一致的格式（`-` 分隔線）

## Risks / Trade-offs

- 風險：若輸入檔案不存在，`os.path.getmtime()` 會拋出例外 → 緩解：程式在此之前已經讀取並解析過這些檔案，若不存在早已在前面報錯
