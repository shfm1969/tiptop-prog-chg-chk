## Context

`chklog.py` 是一個稽核腳本，比對 txt 檔案清單中的程式檔案與 Excel 記錄表（`GP程式修改記錄.xlsx`）。目前報告僅輸出異常（找不到對應）的明細，比對成功時只顯示「全部符合」。使用者希望看到每一筆成功比對的詳細資訊，以便完整追溯。

## Goals / Non-Goals

**Goals:**
- 在四個檢查區塊中，新增比對成功時的明細輸出
- 成功明細格式：`[Row序號][程式編號],[上線日期]`
- Row 序號指 Excel 中的資料列序號（含標題行偏移）
- 保留現有的異常輸出邏輯不變

**Non-Goals:**
- 不修改輸入檔案格式或 Excel 欄位結構
- 不更動異常訊息的格式
- 不新增額外的輸出檔案格式（如 CSV、JSON）

## Decisions

### 1. Row 序號的計算方式
**選擇**: 使用 `DataFrame.iterrows()` 的 index + 3（Excel header 在第2列，pandas 0-based index）來還原 Excel 實際行號。
**理由**: 讓使用者可以直接對照 Excel 檔案中的行號，無須人工換算。

### 2. 成功記錄的收集方式
**選擇**: 新增 `matched_X` 清單（與現有 `missing_X` 並行），在比對成功時記錄 row 序號、程式編號、上線日期。
**理由**: 與現有架構一致，最小化修改範圍。

### 3. 輸出順序
**選擇**: 先輸出成功比對明細，再輸出異常明細。
**理由**: 先呈現正常結果，再列出需關注的異常項目，符合閱讀習慣。

## Risks / Trade-offs

- **輸出檔案變大**: 成功記錄可能大量增加報告行數 → 可接受，因為完整性優先
- **效能影響極小**: 僅增加字串收集與輸出，不影響核心比對邏輯
