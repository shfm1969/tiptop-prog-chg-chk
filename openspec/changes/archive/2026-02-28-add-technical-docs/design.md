## Context

目前專案（用於檢查程式修改紀錄）已經有初步的雛型，包含 `chklog.py`、`check_excel.py` 等腳本。但這些腳本的執行方式、環境設定（如 `uv`）、輸入與輸出資料夾的結構等，並沒有完整的文件紀錄。這會造成後續接手的人員需要花費額外時間閱讀程式碼才能知道如何執行測試或應用。

## Goals / Non-Goals

**Goals:**
- 提供清晰的環境建置指南（包含 Python 環境與依賴套件套件的安裝）。
- 建立主要腳本的使用說明手冊。
- 描述此專案的目錄結構和每個檔案的基本用途。

**Non-Goals:**
- 不涉及應用程式邏輯的修改。
- 不涵蓋 Python 基礎語法的教學。

## Decisions

- **文件格式與位置**：使用 Markdown 格式，並集中在專案根目錄下的 `README.md`，若後續內容過多再考慮拆分到 `docs/` 資料夾。考量到目前專案規模較小，單一的 `README.md` 是最簡單有效的做法。
- **內容結構**：文件將包含以下段落：專案簡介、前置需求（Prerequisites）、環境安裝（Installation）、目錄結構說明（Project Structure）、腳本使用方式（Usage）。

- [Trade-off] 單一檔案容易變得冗長 ??如果文件未來超過 300 行，將開始向外拆分成多個小檔案。
